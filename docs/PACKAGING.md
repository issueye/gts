# GTS 打包说明

## 标准库外部化

从 v0.1.0 开始，GTS 标准库支持外部化部署。这意味着标准库可以作为独立文件部署，而不是编译到二进制中。

### 优势

1. **灵活更新**：无需重新编译即可更新标准库
2. **减小体积**：二进制文件更小
3. **插件化**：标准库可以作为插件加载
4. **开发友好**：便于调试和修改

### 打包步骤

#### 1. 编译核心二进制（不含标准库）

```bash
# 编译最小核心
go build -tags nostdlib -o gs-core.exe ./cmd/gs
```

#### 2. 复制标准库到部署目录

```bash
# 使用脚本自动复制
gs scripts/copy-stdlib.gs ./dist/stdlib

# 或手动复制
cp internal/stdlib/*.go ./dist/stdlib/
```

#### 3. 打包分发

```
dist/
  ├── gs.exe          # 主程序
  └── stdlib/         # 标准库目录
      ├── test.go
      ├── env.go
      ├── json.go
      └── ... (20个模块)
```

### 运行时加载

程序启动时会自动从以下位置查找标准库：

1. `./stdlib/` - 当前目录
2. `<exe_dir>/stdlib/` - 可执行文件目录
3. `$HOME/.gts/stdlib/` - 用户目录

### 兼容性

- ✅ 向后兼容：仍支持静态编译模式
- ✅ 自动降级：找不到外部库时使用内置库
- ✅ 混合模式：部分模块外部，部分内置

### 开发建议

开发时建议使用完整编译模式（包含标准库），生产环境可选择外部化部署以获得更大灵活性。

```bash
# 开发模式（完整）
go build -o gs.exe ./cmd/gs

# 生产模式（外部化）
go build -tags nostdlib -o gs.exe ./cmd/gs
```
